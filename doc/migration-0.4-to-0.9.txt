Necessary steps to migrate applications from MyFaces 0.4.x to 0.9.x
===================================================================

Mandatory steps
---------------
* change <f:use_faces> tags to <f:view>

* replace all key/bundle attribute pairs by the corresponding value binding
    e.g.
        old: <h:output_message bundle="example_messages" key="today" >
        new: <h:output_message value="#{example_messages['today']}" >

* replace all valueRef attributes by the corresponding value binding
    e.g.
        old: <h:output_text valueRef="calcForm.result" />
        new: <h:output_text value="#{calcForm.result}" />

* in all <h:form> tags rename "formName" attributes to "name"

* remove all "commandName" attributes in <h:command_button> and <h:command_link> tags

* replace all actionRef attributes by the corresponding action method binding
    e.g.
        old: <h:command_link actionRef="ucaseForm.jumpHomeAction" >
        new: <h:command_link action="#{ucaseForm.jumpHome}" >
    Note: In this example we must also change the old getJumpHomeAction method to a
          method with the following signature: public String jumpHome()

* rename <x:page_layout> tags to <x:panel_layout> and change attributes:
    layoutRef --> layout with value binding syntax
    panelClass --> styleClass

* rename <x:navigation> tags to <x:panel_navigation> and change attributes:
    bundle --> no longer supported/needed
    panelClass --> styleClass

* rename <x:navigation_item> tags to <x:command_navigation> and change attributes:
    key --> value
    commandClass --> styleClass

* rename <x:navigation_separator> tags to <x:output_navigation>

* rename <h:command_hyperlink> tags to <h:command_link>

* alter <from-view-id> and <to-view-id> values of navigation rules: use .jsp extension instead of .jsf

* rename <h:selectitem> tags to <f:selectitem> and change attributes:
    label --> itemLabel

* rename <x:file_upload> tags to <x:input_fileupload>

* tables: see example simpleList.jsp
          removed: <h:panel_data>
          new tag: <h:column> (containing facet header and footer)
          rename:
            <h:panel_list> --> <h:data_table>
            panel_list@panelClass --> data_table@styleClass
          new attributes:
             panel_list@var
             panel_list@value
          model bean-property: Iterator is not supported anymore, use java.util.List

* rename <x:tabbed_pane> tags to <x:panel_tabbedpane>

* rename <x:tab> tags to <x:panel_tab> and change attributes:
    label --> value

faces-config
------------
* navigation-rules: replace <from-action-ref>x.y</from-action-ref> by <from-action>#{x.y}</from-action>     

Optional steps (deprecated features)
------------------------------------
* The following web.xml init parameters can be removed:
    myfaces_WrapPageContext - did not work with all JSP containers and is obsoltete with Tomcat 5
    myfaces_DefaultLanguage - no longer needed
    myfaces_LogLevel - no longer needed
    myfaces_AutoCreateRequestScopeBeans - no longer needed
    myfaces_ServletMappingMode - no longer needed
    myfaces_DiscardInternalAttributes - no longer needed
